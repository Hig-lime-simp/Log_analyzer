# Log Analyzer - Потоковый анализ логов в реальном времени

Приложение для потокового анализа логов в реальном времени с TUI интерфейсом.

## Возможности

- **Потоковая обработка**: Чтение и обработка логов по мере их появления
- **Многопоточность**: Параллельная обработка несколькими воркерами
- **TUI интерфейс**: Удобный текстовый интерфейс на базе Ratatui
- **Фильтрация**: Фильтрация по уровню, ключевым словам, источнику
- **Статистика**: Агрегированная статистика по уровням, источникам, файлам
- **Обнаружение аномалий**: Детектирование всплесков ошибок и объема
- **Сессии**: Поддержка сессий с возможностью восстановления
- **Конфигурация**: Гибкая настройка через JSON/TOML файлы
- **Отчетность**: Генерация отчетов в файл

## Установка

```bash
# Установите Rust (если не установлен)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Склонируйте репозиторий
git clone <repository-url>
cd log-analyzer

# Соберите проект
cargo build --release

# Запустите
cargo run --release -- [OPTIONS] [FILES...]
```

## Использование

### Базовый запуск

```bash
# Анализ одного файла
./target/release/log-analyzer /var/log/syslog

# Анализ нескольких файлов
./target/release/log-analyzer /var/log/syslog /var/log/auth.log

# С конфигурационным файлом
./target/release/log-analyzer -c config.json /var/log/syslog

# С указанием количества воркеров
./target/release/log-analyzer -w 8 /var/log/syslog

# С сохранением отчета
./target/release/log-analyzer -o report.txt /var/log/syslog

# Восстановление сессии
./target/release/log-analyzer -s <session-id> /var/log/syslog
```

### Параметры командной строки

```
log-analyzer [OPTIONS] [FILES...]

Аргументы:
  [FILES]...  Лог-файлы для анализа

Опции:
  -c, --config <CONFIG>      Путь к конфигурационному файлу (JSON/TOML)
  -s, --session <SESSION>    ID сессии для восстановления
  -w, --workers <WORKERS>    Количество воркеров (по умолчанию: 4)
  -o, --output <OUTPUT>      Путь к файлу отчета
  -h, --help                 Показать справку
  -V, --version              Показать версию
```

## Конфигурация

Пример конфигурационного файла `config.json`:

```json
{
  "rules": [
    {
      "id": "rule-001",
      "name": "Критические ошибки",
      "condition": {
        "type": "Level",
        "value": "Error"
      },
      "action": {
        "type": "Warn",
        "config": {
          "severity": "high"
        }
      },
      "priority": 1,
      "enabled": true
    },
    {
      "id": "rule-002",
      "name": "Ошибки аутентификации",
      "condition": {
        "type": "Keyword",
        "value": "authentication failed"
      },
      "action": {
        "type": "Warn",
        "config": {
          "severity": "critical"
        }
      },
      "priority": 2,
      "enabled": true
    }
  ],
  "workers": 4,
  "time_window_seconds": 60,
  "error_threshold": 10,
  "volume_threshold": 2.0,
  "output": {
    "console": true,
    "report_file": null,
    "report_format": "Text",
    "realtime": true
  }
}
```

### Типы условий (RuleCondition)

- `Level` - фильтрация по уровню лога (Info, Warn, Error, Debug, Trace)
- `Keyword` - поиск по ключевому слову
- `Source` - фильтрация по источнику
- `Pattern` - регулярное выражение
- `ErrorThreshold` - превышение порога ошибок
- `All` - все записи

### Типы действий (RuleAction)

- `Count` - подсчет совпадений
- `Warn` - генерация предупреждения
- `Ignore` - игнорирование записи
- `Tag` - добавление меток
- `Forward` - пересылка во внешнюю систему

## Интерфейс

### Вкладки

1. **Logs (1)** - Поток логов в реальном времени
2. **Statistics (2)** - Статистика по файлам и уровням
3. **Anomalies (3)** - Обнаруженные аномалии

### Управление

- `q` - Выход
- `Tab` / `1`, `2`, `3` - Переключение вкладок
- `↑` `↓` - Прокрутка логов
- `/` - Поиск (планируется)

### Цветовая индикация уровней

- 🔴 ERROR - Красный
- 🟡 WARN - Желтый
- 🟢 INFO - Зеленый
- 🔵 DEBUG - Синий
- 🟣 TRACE - Фиолетовый

## Технические требования

- **Rust** с использованием:
  - `ratatui` + `crossterm` - TUI интерфейс
  - `tokio` - асинхронная runtime
  - `dashmap` - потокобезопасные хэш-мапы
  - `notify` - мониторинг файлов
  - `serde` + `toml` + `serde_json` - сериализация
  - `clap` - парсинг CLI аргументов
  - `chrono` - работа со временем
  - `regex` - регулярные выражения
  - `uuid` - генерация ID сессий

## Безопасность и обработка ошибок

- Ошибки не приводят к падению программы
- Возврат структурированных ошибок через `thiserror`
- Graceful shutdown всех потоков при остановке
- Потокобезопасные структуры данных (`Arc`, `DashMap`)
- Обработка исключений с информированием пользователя
