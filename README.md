# Log Analyzer

Веб-приложение для анализа логов в реальном времени, написанное на Rust.

## Описание

Log Analyzer — это серверное приложение с веб-интерфейсом, которое позволяет:
- Добавлять логи через API
- Просматривать все сохранённые логи в виде таблицы
- Фильтровать логи по уровням (INFO, WARNING, ERROR)
- Получать статистику по логам
- Обновлять данные в реальном времени через WebSocket

## Технологии

- **Backend**: Rust + Axum (веб-фреймворк)
- **Хранение данных**: DashMap (потокобезопасная хэш-таблица в памяти)
- **Real-time обновления**: WebSocket
- **Frontend**: HTML, CSS, Vanilla JavaScript

## Установка и запуск

### Требования

- Rust (версия 2021 или новее)
- Cargo

### Запуск

```bash
# Сборка и запуск приложения
cargo run --release
```

После запуска сервер будет доступен по адресу: **http://localhost:3000**

## API

### GET `/api/logs`
Возвращает список всех сохранённых логов.

**Ответ:**
```json
[
  {
    "id": 0,
    "content": "[ERROR] Database connection failed",
    "level": "error",
    "timestamp": "2024-01-15T10:30:00Z"
  }
]
```

### POST `/api/logs`
Добавляет новый лог.

**Тело запроса:**
```json
"Текст лога"
```

**Ответ:**
```json
{
  "id": 1,
  "content": "Текст лога",
  "level": "info",
  "timestamp": "2024-01-15T10:35:00Z"
}
```

### GET `/api/stats`
Возвращает статистику по логам.

**Ответ:**
```json
{
  "total": 100,
  "errors": 5,
  "warnings": 10,
  "info": 85
}
```

### WebSocket `/ws`
Подключение к WebSocket для получения обновлений в реальном времени. При добавлении нового лога всем подключённым клиентам отправляется сообщение с данными лога.

## Функционал веб-интерфейса

1. **Ввод лога**: Текстовое поле для ввода текста лога и кнопка отправки
2. **Статистика**: Карточки с количеством логов по уровням (Всего, INFO, WARNING, ERROR)
3. **Фильтрация**: Кнопки для фильтрации логов по уровням
4. **Таблица логов**: Отображение времени, уровня и содержания каждого лога
5. **Real-time обновления**: Новые логи появляются автоматически без перезагрузки страницы

## Определение уровня лога

Уровень лога определяется автоматически по содержанию:
- `[ERROR]` или `error` → ERROR
- `[WARNING]`, `[WARN]` или `warning`, `warn` → WARNING
- Всё остальное → INFO

## Зависимости

- `dashmap` — потокобезопасная хэш-таблица
- `chrono` — работа с датой и временем
- `tower-http` — middleware для раздачи статических файлов
- `futures-util` — утилиты для работы с futures
